# 🗺️ Roadmap Personalizado: Analista de Dados (Data Analyst)

**Disponibilidade:** 14 a 21 horas semanais

**Experiência:** Zero (buscando primeira oportunidade na área)

**Objetivo:** Primeiro emprego

**Foco:** Dados e código

---

## 🧩 VISÃO DO DIA A DIA
Como é o trabalho de uma Analista de Dados:
- Coletar, limpar e organizar grandes volumes de dados de diferentes fontes.
- Criar dashboards interativos e relatórios visuais para facilitar a tomada de decisão.
- Escrever consultas em bancos de dados para extrair e cruzar informações específicas.
- Identificar padrões estatísticos e tendências de mercado ou comportamento de usuários.
- Apresentar os resultados de forma clara e didática para gestores e tomadores de decisão.

---

## 🧠 MAPA DE SKILLS

**CORE SKILLS (essenciais):**
- Lógica de Programação e Estruturas de Dados
- Versionamento de Código (Git & GitHub)
- SQL Avançado e Python para Análise de Dados

**NICE-TO-HAVE (complementares):**
- Storytelling com Dados (comunicação eficaz)
- Noções de Engenharia de Dados (ETL básico)
- Experiência em Cloud (AWS ou Azure)

**FERRAMENTAS E TECNOLOGIAS:**
- Git, GitHub e VS Code
- Python (Pandas, NumPy, Plotly)
- Bancos de Dados Relacionais (PostgreSQL, MySQL)
- Power BI ou Google Looker Studio

## 📅 ROADMAP DE 90 DIAS

### MÊS 1 - FUNDAMENTOS E VERSIONAMENTO
**SEMANA 1-2:**
- **Git & GitHub:** Instalação, configuração e domínio do fluxo (clone, add, commit, push, pull).
- **Lógica e Python:** Sintaxe básica, tipos de dados e estruturas de decisão focadas em processamento de listas.

**SEMANA 3-4:**
- **Python para Dados:** Manipulação de tabelas com Pandas e limpeza de dados (tratar valores nulos e duplicados).
- **GitHub Profissional:** Organização do perfil, criação de repositórios e escrita de READMEs atrativos.

### MÊS 2 - SQL E ESTRUTURAÇÃO
**SEMANA 5-6:**
- **Bancos de Dados:** Instalação do PostgreSQL e domínio de comandos DDL (CREATE, ALTER) e DML (SELECT, INSERT).
- **SQL para Analistas:** Filtros complexos, funções de agregação e subqueries.

**SEMANA 7-8:**
- **Relacionamentos:** Domínio absoluto de JOINs e entendimento de modelagem dimensional (Star Schema).
- **Integração:** Como conectar o Python ao SQL para extrair dados diretamente do banco.

### MÊS 3 - PORTFÓLIO DE IMPACTO E CARREIRA
**SEMANA 9-10:**
- **Desenvolvimento do Projeto:** Execução da análise de dados do projeto de portfólio "Business Intelligence".
- **Visualização:** Construção do dashboard final com foco em KPIs (indicadores de desempenho).

**SEMANA 11-12:**
- **Documentação:** Criar um artigo ou post no LinkedIn detalhando os insights encontrados no projeto.
- **Preparação:** Simulação de entrevistas técnicas e refinamento do currículo com foco na graduação em Ciência da Computação.

## 🚀 PROJETO DE PORTFÓLIO

**PROJETO: Dashboard de Otimização de Vendas e Previsão de Estoque (Retail Intelligence)**

**O QUE FAZER:**
Simular um cenário real de uma loja de varejo. Analisar o histórico de vendas, identificar quais produtos dão mais lucro, quais estão parados no estoque e sugerir quando comprar novos produtos para evitar perdas financeiras.

**ENTREGÁVEIS:**
- Repositório no GitHub com o script de limpeza em Python.
- Banco de dados SQL com as tabelas de vendas, produtos e clientes relacionadas.
- Dashboard interativo com os principais indicadores financeiros.

**CRITÉRIOS DE ACEITAÇÃO:**
- O dashboard deve mostrar o Ticket Médio por cliente e a Margem de Lucro por categoria.
- Deve haver uma análise de "Churn" (clientes que pararam de comprar).
- O código deve estar versionado com mensagens de commit claras e profissionais.

## 💬 ROTEIRO DE PREPARAÇÃO PARA ENTREVISTAS

**PERGUNTA 1: Como você organiza seus projetos e garante que seu código não seja perdido?**
**COMO RESPONDER:** Explique seu fluxo de trabalho usando Git e GitHub. Diga que utiliza commits atômicos e descritivos para manter o histórico de alterações e que organiza o GitHub para que qualquer recrutador consiga entender a lógica do projeto apenas lendo a documentação do README.

**PERGUNTA 2: Por que você escolheu Ciência da Computação e como isso te ajuda na análise de dados?**
**COMO RESPONDER:** Destaque que o curso te dá uma base sólida em algoritmos e estrutura de dados, o que te permite escrever códigos mais performáticos e entender melhor como os dados são processados por trás das ferramentas de visualização.

**PERGUNTA 3: Em um banco de dados, qual a diferença entre um INNER JOIN e um LEFT JOIN?**
**COMO RESPONDER:** "O INNER JOIN retorna apenas os registros que possuem correspondência em ambas as tabelas. Já o LEFT JOIN retorna todos os registros da tabela da esquerda e os correspondentes da direita; se não houver par, ele traz valores nulos."

**PERGUNTA 4: Como você lidaria com uma base de dados extremamente 'suja' e desorganizada?**
**COMO RESPONDER:** Diga que usaria Python (Pandas) para identificar outliers, tratar dados nulos e padronizar formatos de data e texto, garantindo a integridade da análise antes de levar os dados para o dashboard.

**PERGUNTA 5: Qual o seu objetivo de carreira até a graduação em 2030?**
**COMO RESPONDER:** Mostre que você quer crescer na área de dados, começando como Analista Júnior para dominar o negócio e, gradualmente, evoluir para Ciência de Dados ou Engenharia de Dados, utilizando toda a base técnica que a faculdade e o mercado vão proporcionar.
